What is state management

As your application grows, it helps to be more intentional about how your state is organized and how the data flows between your components. Redundant or duplicate state is a common source of bugs.
State management is simply a way to organize communication and sharing of data across components.

React useState

Consider you have made a react app to manage your trainings at home.

React useState

              <Training>

                state = useState
              
                <TrainingProgressTable state />

              </Training>

Since all its code is written in a single file and it consists of only two components you don't really need any state management library. The state is being kept by the useState hook inside the main component.

React useState

    App features:
  • list of exercises
  • a timer between each approach and between each exercise to take a rest
  • feature to track the number of completed approaches for each exercise
  • an integrated stopwatch
  • an ability to save the duration of each exercise somewhere

The app was written really fast so after some time it becomes hard to support. You want to add some new features like running a timer for some exercises and keep the duration of each exercise somewhere.

Props Drilling


              <Training>
                state = useState
                <TrainingReader state onStateUpdate />
                <TrainingProgressTable state onStateUpdate />  
              </Training>
              
              <TrainingProgressTable state onStateUpdate>  
                <ExerciseStopwatch exercise onStateUpdate />
                <TrainingSender state />
                <TrainingTimer state />
              <TrainingProgressTable />
              
              <TrainingSender state>
                <SomethingElse state>
              <TrainingSender />
              
              <SomethingElse state>
                <Whatever state />
              <SomethingElse />

So you break it up into several components. The app is growing and the number of components is increasing. After some time you need to pass the props through several components that are nested one into the other. It is called props drilling.

What is it Redux?

Redux is a state container for js application.

It can be used in pair with React or any other view library.

History of Redux

Redux was created by Dan Abramov and Andrew Clark in 2015. Abramov began writing the first Redux implementation while preparing for a conference talk at React Europe on hot reloading. Abramov remarks, "I was trying to make a proof of concept of Flux where I could change the logic. And it would let me time travel. And it would let me reapply the future actions on the code change.

Redux Data Flow​

Three Principles

  1. Single Source of Truth
  2. State is read-only
  3. Changes are made with pure functions
1. The global state of your application is stored in an object tree within a single store.

2. The only way to change the state is to emit an action, an object describing what happened.

3. To specify how the state tree is transformed by actions, you write pure reducers.
